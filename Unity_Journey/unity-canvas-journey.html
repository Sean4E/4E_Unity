01

Create a Canvas

Every UI element in Unity must be a child of a Canvas. No Canvas = no UI on screen.

The Canvas is the browser window. Without it, your HTML elements have nowhere to render.
How to add: Right-click Hierarchy → UI → Canvas. Unity auto-creates an EventSystem too — don't delete it!

Always set Canvas Scaler to "Scale With Screen Size" with 1920 x 1080. Otherwise your UI will look wrong on different screens.

Three render modes:

Screen Space — Overlay (most common)
UI drawn on top of everything. No camera needed. Use for HUDs, menus, scores.

Screen Space — Camera
UI tied to a camera. Allows post-processing on UI.

World Space
Canvas as a 3D object in the scene. Use for in-game displays, VR/AR interfaces, floating nametags.

02

Text & Labels

Use TextMeshPro (TMP) for all text. Unity's old Text component is blurry — TMP is sharp at any size.

First time? Unity will ask you to "Import TMP Essentials" — click yes. This loads fonts and shaders.
TextMeshPro is like using a web font with anti-aliasing — crisp at every size, supports rich text markup like <b>, <color>, <size>.

Add text: Right-click Canvas in Hierarchy → UI → Text - TextMeshPro

Access TMP from a C# script:

C# — Change text at runtime 
using TMPro;
using UnityEngine;

public class ScoreDisplay : MonoBehaviour
{
    [SerializeField] TMP_Text scoreText;
    int score = 0;

    public void AddPoint()
    {
        score++;
        scoreText.text = "Score: " + score;
    }
}
TMP_Text works for both UI text and world-space 3D text.

OnClick event — like addEventListener('click'). In Unity, you wire it in the Inspector or with code:

C# — Button click listener 
using UnityEngine;
using UnityEngine.UI;

public class MenuController : MonoBehaviour
{
    [SerializeField] Button startButton;

    void Start()
    {
        // Same as: startButton.addEventListener("click", StartGame)
        startButton.onClick.AddListener(StartGame);
    }

    void StartGame()
    {
        Debug.Log("Game Started!");
    }
}

Image Type: Filled is how health bars work — the fill amount goes from 0 to 1. We'll use this in Step 08.

Import sprites: Drag an image into your Assets folder. In the Inspector, set Texture Type to Sprite (2D and UI), then click Apply.

Four Image Types:

Simple — stretches the sprite to fill the rect
Sliced — 9-slice scaling (borders stay crisp, center stretches)

Tiled — repeats the sprite to fill the area
Filled — reveals with a fill amount (0 to 1)

Panels & Layouts

Layout Groups are CSS Flexbox. Vertical = flex-direction:column. Horizontal = flex-direction:row. Grid = display:grid.

Sliders & Toggles

A Slider lets the player pick a number in a range. A Toggle is an on/off switch.

Slider = <input type="range">. Toggle = <input type="checkbox">. Same idea, different look.
They're visual only — scripts connect them to actual behaviour. A slider does nothing until you wire an OnValueChanged event.

Slider anatomy: Background, Fill Area → Fill, Handle Slide Area → Handle.

C# — Slider + Toggle listeners 
using UnityEngine;
using UnityEngine.UI;
using TMPro;

public class SettingsUI : MonoBehaviour
{
    [SerializeField] Slider volumeSlider;
    [SerializeField] Toggle musicToggle;
    [SerializeField] TMP_Text volumeLabel;

    void Start()
    {
        volumeSlider.onValueChanged.AddListener(OnVolumeChanged);
        musicToggle.onValueChanged.AddListener(OnMusicToggled);
    }

    void OnVolumeChanged(float value)
    {
        volumeLabel.text = "Volume: " + Mathf.RoundToInt(value);
        AudioListener.volume = value / 100f;
    }

    void OnMusicToggled(bool isOn)
    {
        Debug.Log("Music: " + (isOn ? "ON" : "OFF"));
    }
}

C# — Health bar script 
using UnityEngine;
using UnityEngine.UI;
using TMPro;

public class HealthBar : MonoBehaviour
{
    [SerializeField] Image fillImage;
    [SerializeField] TMP_Text hpText;

    int maxHP = 100;
    int currentHP;

    void Start()
    {
        currentHP = maxHP;
        UpdateBar();
    }

    public void TakeDamage(int amount)
    {
        currentHP = Mathf.Max(0, currentHP - amount);
        UpdateBar();
    }

    public void Heal(int amount)
    {
        currentHP = Mathf.Min(maxHP, currentHP + amount);
        UpdateBar();
    }

    void UpdateBar()
    {
        float ratio = (float)currentHP / maxHP;
        fillImage.fillAmount = ratio;
        hpText.text = "HP: " + currentHP + " / " + maxHP;

        // Color shifts: green -> yellow -> red
        if (ratio > 0.5f)
            fillImage.color = Color.Lerp(
                Color.yellow, Color.green,
                (ratio - 0.5f) * 2f);
        else
            fillImage.color = Color.Lerp(
                Color.red, Color.yellow,
                ratio * 2f);
    }
}
You built a health bar! This same pattern works for mana bars, stamina bars, XP bars, loading bars — anything that fills.

Pause Menu

A pause menu is a Panel that covers the screen. Press Escape → show it. Click Resume → hide it. Time stops when paused.

Like a modal dialog in HTML — an overlay div that blocks interaction with the page underneath.

C# — Pause toggle script 
using UnityEngine;

public class PauseMenu : MonoBehaviour
{
    [SerializeField] GameObject pausePanel;
    bool isPaused = false;

    void Update()
    {
        if (Input.GetKeyDown(KeyCode.Escape))
            TogglePause();
    }

    public void TogglePause()
    {
        isPaused = !isPaused;
        pausePanel.SetActive(isPaused);

        // Freeze / unfreeze time
        Time.timeScale = isPaused ? 0f : 1f;

        // Show cursor when paused
        Cursor.lockState = isPaused
            ? CursorLockMode.None
            : CursorLockMode.Locked;
    }

    public void ResumeGame()
    {
        isPaused = true; // flipped to false inside
        TogglePause();
    }

    public void QuitGame()
    {
        Time.timeScale = 1f;
        Application.Quit();
    }
}
Time.timeScale = 0 freezes all physics and time-based animations. Always set it back to 1 when resuming!

Anchoring & Canvas Group

Anchors control where a UI element stays when the screen resizes. Get them wrong and your UI breaks on different resolutions.

Anchors are like CSS position: fixed combined with top/left/right/bottom. They pin your element to a part of the screen.

Canvas Group is the easiest way to fade a panel in or out. Animate the Alpha from 0 to 1.
C# — Fade coroutine using Canvas Group 
using System.Collections;
using UnityEngine;

public class UIFader : MonoBehaviour
{
    [SerializeField] CanvasGroup panel;
    [SerializeField] float fadeDuration = 0.3f;

    public void FadeIn()
    {
        panel.gameObject.SetActive(true);
        StartCoroutine(Fade(0f, 1f));
    }

    public void FadeOut()
    {
        StartCoroutine(FadeAndDisable());
    }

    IEnumerator Fade(float from, float to)
    {
        float elapsed = 0f;
        while (elapsed < fadeDuration)
        {
            elapsed += Time.unscaledDeltaTime;
            panel.alpha = Mathf.Lerp(from, to,
                elapsed / fadeDuration);
            yield return null;
        }
        panel.alpha = to;
        panel.interactable = to > 0.5f;
        panel.blocksRaycasts = to > 0.5f;
    }

    IEnumerator FadeAndDisable()
    {
        yield return Fade(1f, 0f);
        panel.gameObject.SetActive(false);
    }
}
Time.unscaledDeltaTime keeps the fade working even when Time.timeScale = 0 (game is paused).
