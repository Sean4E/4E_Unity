01

Project Setup

Open Unity Hub and create a new project. We'll use the 3D (URP) template for better visuals and the New Input System for modern input handling.

  1. Open Unity Hub → Click New Project
  2. Select 3D (URP) template
  3. Name your project (e.g. ThirdPersonController)
  4. Choose a save location
  5. Click Create project and wait for Unity to open

Install the New Input System

Unity 6 may already have it installed. If not:

  1. Go to Window → Package Manager
  2. In the top-left dropdown, select Unity Registry
  3. Search for Input System
  4. Click Install
  5. Unity will ask to restart — click Yes
Important: After installing, go to Edit → Project Settings → Player → Other Settings. Under Active Input Handling, select Both or Input System Package (New). Unity needs a restart after this change.

Organise Your Folders

In the Project window, right-click in the Assets folder and create these folders:

Assets/
Scripts/ — C# controller + camera scripts
Animations/ — Animator controllers + animation clips
Models/ — Character FBX + animation FBX files
Materials/ — Ground, character materials

Where to Get a Character + Animations (Free)

▶

Mixamo (mixamo.com) is Adobe's free service that provides rigged characters and animations. You'll need a free Adobe account.

  1. Go to mixamo.com and sign in with an Adobe account
  2. Click Characters tab → pick any character you like
  3. Download the character: Format = FBX for Unity (.fbx), Pose = T-Pose
  4. Click Animations tab → search for "Idle"
  5. Select an idle animation → click Download: Format = FBX for Unity, Skin = Without Skin
  6. Repeat for "Walking" and "Running"
You should end up with 4 FBX files: one character model (with skin), and three animation files (without skin). Drag all of them into your Assets/Models folder.
Checkpoint: You have a URP project with organised folders, the New Input System installed, and 4 FBX files (character + 3 animations) in your Models folder.

02

Character Setup

Now we configure the FBX files so Unity understands the skeleton (rig) and can retarget animations between different characters.

Configure the Character Model

  1. In the Project window, click on your character FBX file (the one with the mesh/skin)
  2. In the Inspector, click the Rig tab
  3. Set Animation Type to Humanoid
  4. Click Apply at the bottom
  5. Click Configure... to verify the bone mapping (green dots = good)
  6. If all bones are green, click Done

Why Humanoid? Unity's Humanoid system maps any character's bones to a standard skeleton. This means you can use the same walk animation on any humanoid character — tall, short, robot, alien. It's like CSS classes — one set of rules, many elements.

Configure the Animation FBX Files

Do this for each animation file (Idle, Walk, Run):

  1. Click the animation FBX file in the Project window
  2. Go to the Rig tab → set Animation Type to Humanoid
  3. Set Avatar Definition to Copy From Other Avatar
  4. Drag your character's Avatar into the Source slot (find it inside the character FBX — expand the arrow)
  5. Click Apply
  6. Go to the Animation tab
  7. Tick Loop Time (all three animations should loop)
  8. Click Apply

Place the Character in the Scene

  1. Drag the character FBX from the Project window into the Scene
  2. Set Position to X: 0, Y: 0, Z: 0
  3. Make sure the character is standing on the ground (feet at Y = 0)
  4. Rename the GameObject to Player
Common issue: If the character is too small or too big, select the character FBX in the Project window, go to the Model tab, and adjust the Scale Factor. Click Apply. A Mixamo character at scale 1 is usually fine.

Animator Controller

Create a state machine that blends between Idle, Walk, and Run based on movement speed.

03

Animator Controller

The Animator Controller is a visual state machine. Each state plays an animation clip, and transitions define when to switch between them. Think of it like CSS media queries — "when this condition is true, switch to this state".

Create the Animator Controller

  1. In the Project window, navigate to your Assets/Animations folder
  2. Right-click → Create → Animator Controller
  3. Name it PlayerAnimator
  4. Double-click it to open the Animator window

Add States

  1. Find your Idle animation clip (expand the animation FBX in the Project window — click the triangle arrow — you'll see the clip inside)
  2. Drag the Idle clip into the Animator window — it becomes an orange state (default)
  3. Drag the Walk clip into the Animator window — it becomes a grey state
  4. Drag the Run clip into the Animator window — another grey state
The orange state is the default (Entry state). Idle should be orange because the character starts standing still. If Walk or Run became the default, right-click on Idle and select Set as Layer Default State.

Add Parameters

  1. In the Animator window, click the Parameters tab (top-left)
  2. Click the + button → select Float
  3. Name the parameter Speed (case sensitive!)
Parameters are like JavaScript variables that your script writes to and the Animator reads from. When your code sets Speed = 0.5, the Animator knows the character is walking.

Create Transitions

Right-click a state and select Make Transition, then click the destination state. Create these four transitions:

FromToCondition
IdleWalkSpeed Greater than 0.1
WalkIdleSpeed Less than 0.1
WalkRunSpeed Greater than 0.9
RunWalkSpeed Less than 0.9

How to Create Each Transition

  1. Right-click the Idle state → Make Transition
  2. Click on the Walk state (an arrow appears)
  3. Click the arrow (the transition line) to select it
  4. In the Inspector, uncheck Has Exit Time
  5. Set Transition Duration to 0.1
  6. Under Conditions, click +
  7. Set: Speed — Greater — 0.1
  8. Repeat for all four transitions

Critical: Always uncheck Has Exit Time for responsive transitions. With it enabled, the character waits for the current animation to finish before transitioning — it feels sluggish and unresponsive.

Core Scripts
Movement & Camera

Two C# scripts: one for player movement and animation, one for camera orbit.

04

Third Person Controller Script

This script handles movement, rotation, gravity, and animation. It reads WASD/Arrow keys, moves the character relative to the camera direction, and tells the Animator how fast the character is moving.

Create the Script

  1. In the Project window, navigate to Assets/Scripts
  2. Right-click → Create → C# Script
  3. Name it ThirdPersonController (exact spelling, no spaces)
  4. Double-click to open it in your code editor
  5. Delete everything and paste the code below
  6. Save the file (Ctrl+S)
C# — ThirdPersonController.cs 
using UnityEngine;
using UnityEngine.InputSystem;

public class ThirdPersonController : MonoBehaviour
{
    [Header("Movement")]
    public float walkSpeed = 3f;
    public float runSpeed = 6f;
    public float rotationSpeed = 10f;
    public float gravity = -9.81f;

    private CharacterController controller;
    private Animator animator;
    private Transform cameraTransform;
    private float verticalVelocity;

    void Start()
    {
        controller = GetComponent<CharacterController>();
        animator = GetComponent<Animator>();
        cameraTransform = Camera.main.transform;
    }

    void Update()
    {
        if (Keyboard.current == null) return;

        // Read input
        float x = 0f, z = 0f;
        if (Keyboard.current.aKey.isPressed ||
            Keyboard.current.leftArrowKey.isPressed) x -= 1f;
        if (Keyboard.current.dKey.isPressed ||
            Keyboard.current.rightArrowKey.isPressed) x += 1f;
        if (Keyboard.current.wKey.isPressed ||
            Keyboard.current.upArrowKey.isPressed) z += 1f;
        if (Keyboard.current.sKey.isPressed ||
            Keyboard.current.downArrowKey.isPressed) z -= 1f;

        Vector3 input = new Vector3(x, 0f, z).normalized;
        bool isRunning = Keyboard.current.leftShiftKey.isPressed;
        float speed = isRunning ? runSpeed : walkSpeed;

        if (input.magnitude > 0.1f)
        {
            // Move relative to camera
            float targetAngle =
                Mathf.Atan2(input.x, input.z)
                * Mathf.Rad2Deg
                + cameraTransform.eulerAngles.y;
            Quaternion targetRotation =
                Quaternion.Euler(0f, targetAngle, 0f);
            transform.rotation = Quaternion.Slerp(
                transform.rotation,
                targetRotation,
                rotationSpeed * Time.deltaTime);

            Vector3 moveDir =
                Quaternion.Euler(0f, targetAngle, 0f)
                * Vector3.forward;
            controller.Move(moveDir * speed * Time.deltaTime);
        }

        // Gravity
        if (controller.isGrounded && verticalVelocity < 0f)
            verticalVelocity = -2f;
        verticalVelocity += gravity * Time.deltaTime;
        controller.Move(Vector3.up
            * verticalVelocity * Time.deltaTime);

        // Animator
        float animSpeed =
            input.magnitude * (isRunning ? 1f : 0.5f);
        animator.SetFloat("Speed",
            animSpeed, 0.1f, Time.deltaTime);
    }
}

The key insight: input.magnitude is 0 when standing still and 1 when moving. Multiply by 0.5 for walking (Speed = 0.5) or by 1.0 for running (Speed = 1.0). The Animator transitions fire at 0.1 and 0.9 — matching these values perfectly.

05

Camera Follow Script

This script makes the camera orbit around the player using mouse input. It follows the character smoothly, stays at a fixed distance, and lets you look around freely.

Create the Script

  1. In Assets/Scripts, right-click → Create → C# Script
  2. Name it ThirdPersonCamera
  3. Double-click to open, delete everything, and paste the code below
  4. Save (Ctrl+S)
C# — ThirdPersonCamera.cs 
using UnityEngine;
using UnityEngine.InputSystem;

public class ThirdPersonCamera : MonoBehaviour
{
    public Transform target;
    public float distance = 5f;
    public float height = 2f;
    public float sensitivity = 2f;
    public float smoothSpeed = 10f;
    public float minVerticalAngle = -20f;
    public float maxVerticalAngle = 60f;

    private float yaw;
    private float pitch = 10f;

    void Start()
    {
        Cursor.lockState = CursorLockMode.Locked;
        Cursor.visible = false;
        if (target != null)
            yaw = target.eulerAngles.y;
    }

    void LateUpdate()
    {
        if (target == null ||
            Mouse.current == null) return;

        // Mouse orbit
        yaw += Mouse.current.delta.x.ReadValue()
            * sensitivity * 0.1f;
        pitch -= Mouse.current.delta.y.ReadValue()
            * sensitivity * 0.1f;
        pitch = Mathf.Clamp(pitch,
            minVerticalAngle, maxVerticalAngle);

        // Calculate position
        Quaternion rotation =
            Quaternion.Euler(pitch, yaw, 0f);
        Vector3 offset =
            rotation * new Vector3(0f, 0f, -distance)
            + new Vector3(0f, height, 0f);
        Vector3 targetPos =
            target.position + offset;

        // Smooth follow
        transform.position = Vector3.Lerp(
            transform.position, targetPos,
            smoothSpeed * Time.deltaTime);
        transform.LookAt(
            target.position
            + Vector3.up * 1.5f);

        // Escape to unlock cursor
        if (Keyboard.current != null &&
            Keyboard.current.escapeKey
                .wasPressedThisFrame)
        {
            Cursor.lockState =
                CursorLockMode.None;
            Cursor.visible = true;
        }

        // Click to re-lock cursor
        if (Mouse.current.leftButton
                .wasPressedThisFrame &&
            Cursor.lockState !=
                CursorLockMode.Locked)
        {
            Cursor.lockState =
                CursorLockMode.Locked;
            Cursor.visible = false;
        }
    }
}

How the Camera Works (click to expand)

▶
ConceptExplanation
LateUpdateRuns after Update() — so the character moves first, then the camera follows. Prevents jitter.
yaw / pitchYaw = left-right rotation. Pitch = up-down tilt. Together they define where the camera looks.
Quaternion.Euler(pitch, yaw, 0)Converts the two angles into a rotation.
rotation * (0, 0, -distance)Points backward from the rotation by the distance amount — positions the camera behind.
+ (0, height, 0)Lifts the camera up so it looks down at the character slightly.
Vector3.LerpSmooth interpolation — camera doesn't teleport, it glides to the target position.
LookAt(target + up * 1.5f)Camera aims at the character's chest/shoulder area, not the feet.
CursorLockMode.LockedHides and locks the cursor to the centre of the screen for FPS-style mouse look.
Collision avoidance: For a more polished camera, you can add a Raycast from the target to the camera position. If it hits a wall, move the camera closer. This is an optional advanced feature.

Polish & Extras

Your controller works. Now let's talk about ways to take it further. These are optional enhancements — try them if you have time.

Blend Trees (Smoother Animation Blending)

▶

Instead of separate states with transitions, a Blend Tree smoothly blends between Idle, Walk, and Run based on the Speed parameter. The result is butter-smooth transitions with no "pop" between states.

  1. In the Animator window, right-click → Create State → From New Blend Tree
  2. Double-click the Blend Tree to enter it
  3. Set the Parameter to Speed
  4. Click + → Add Motion Field (three times)
  5. Assign: Idle (threshold 0), Walk (threshold 0.5), Run (threshold 1.0)
  6. Delete the old Idle, Walk, Run states and their transitions
  7. Set the Blend Tree as the default state
A Blend Tree is like CSS transition: all 0.3s ease — it smoothly interpolates between states instead of snapping. One node replaces three states and four transitions.

Jump (Adding a Jump Animation)

▶

To add jumping, you need a jump animation from Mixamo and a few additions to the controller script.

  1. Download a Jump animation from Mixamo (without skin, Loop Time OFF)
  2. In the Animator, add a Bool parameter called IsJumping
  3. Create a Jump state with the jump animation
  4. Add transitions: Any State → Jump (IsJumping = true), Jump → Idle (IsJumping = false)
  5. In the controller script, add:
// Add to the Update() method after the gravity section:
if (controller.isGrounded &&
    Keyboard.current.spaceKey.wasPressedThisFrame)
{
    verticalVelocity = Mathf.Sqrt(2f * -gravity);
    animator.SetBool("IsJumping", true);
}

if (controller.isGrounded && verticalVelocity < 0f)
{
    animator.SetBool("IsJumping", false);
}

Ground Check (More Reliable Detection)

▶

controller.isGrounded can sometimes be unreliable. A more robust approach uses a small sphere check at the character's feet.

// Add these fields to the class:
public Transform groundCheck;
public float groundDistance = 0.2f;
public LayerMask groundMask;

// Replace controller.isGrounded with:
bool isGrounded = Physics.CheckSphere(
    groundCheck.position, groundDistance, groundMask);
  1. Create an Empty GameObject as a child of the Player, name it GroundCheck
  2. Position it at the character's feet: Y: 0
  3. Drag it into the Ground Check slot on the script
  4. Create a Layer called Ground and assign it to your ground objects
  5. Set the Ground Mask to the Ground layer

Cinemachine (Unity's Built-in Camera System)

▶

Cinemachine is Unity's professional camera system. It can replace our ThirdPersonCamera script with a more polished, feature-rich solution — including collision avoidance, noise/shake, and smooth transitions between camera angles.

  1. Install Cinemachine from Window → Package Manager
  2. Delete the ThirdPersonCamera script from your Main Camera
  3. Select Main Camera → Add Component → CinemachineBrain
  4. Right-click Hierarchy → Cinemachine → Third Person Follow
  5. Set the Follow and Look At targets to your Player
  6. Adjust the shoulder offset, distance, and damping to taste
Cinemachine is what professional games use. Our custom script teaches you the fundamentals — Cinemachine applies the same concepts with more features and polish.
