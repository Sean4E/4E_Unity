What is AR?

Augmented Reality overlays digital content on the real world through your phone's camera. You see your room — but with virtual objects sitting on your desk, your floor, your walls.

AR Foundation is like Three.js but the camera is your phone, the scene is the real world, and the ground plane is your actual floor.

Three types of AR tracking:

Marker-based (Image Tracking)
Detects a specific image (poster, QR code) and pins content to it.

Markerless (Plane Detection)
Detects flat surfaces (floors, tables, walls) and lets you place objects. This is what we'll build.

Face Tracking
Detects faces and overlays filters, masks, or effects. Used in Snapchat and Instagram filters.

AR Foundation
Unity's cross-platform AR framework. One codebase → works on ARCore (Android) + ARKit (iOS).

AR Session

Every AR app needs two things: an AR Session (manages the AR lifecycle) and an XR Origin (the real-world origin point for your virtual content).

AR Session is like renderer.setAnimationLoop() — it runs the AR loop. XR Origin is like scene.position — the origin of your virtual world relative to the real world.

Plane Detection

The AR Plane Manager detects flat surfaces in the real world — floors, tables, walls. Once detected, they appear as mesh outlines you can place objects on.

Like auto-detecting the ground in Three.js, but the "ground" is your actual physical floor. The phone's sensors do the heavy lifting.
Add it to: Select XR Origin → Add Component → AR Plane Manager

Log detected planes with a simple script:

C# — PlaneLogger.cs 
using UnityEngine;
using UnityEngine.XR.ARFoundation;

public class PlaneLogger : MonoBehaviour
{
    ARPlaneManager planeManager;

    void Awake() => planeManager = GetComponent<ARPlaneManager>();

    void OnEnable() => planeManager.planesChanged += OnPlanesChanged;
    void OnDisable() => planeManager.planesChanged -= OnPlanesChanged;

    void OnPlanesChanged(ARPlanesChangedEventArgs args)
    {
        foreach (var plane in args.added)
            Debug.Log($"New plane: {plane.alignment}");
    }
}
04

Place Objects

Now the magic: tap a detected surface and a virtual object appears there. This uses an AR Raycast to find where your finger is pointing in the real world.

Like raycaster.intersectObjects() in Three.js, but the ray goes from your screen tap through the camera into the real world and hits a detected plane.

Add AR Raycast Manager to XR Origin, then create this script:

C# — PlaceObject.cs 
using System.Collections.Generic;
using UnityEngine;
using UnityEngine.XR.ARFoundation;
using UnityEngine.XR.ARSubsystems;

public class PlaceObject : MonoBehaviour
{
    public GameObject prefab;
    ARRaycastManager raycastManager;
    static List<ARRaycastHit> hits = new List<ARRaycastHit>();
    GameObject placed;

    void Awake() => raycastManager = GetComponent<ARRaycastManager>();

    void Update()
    {
        if (Input.touchCount == 0) return;
        Touch touch = Input.GetTouch(0);
        if (touch.phase != TouchPhase.Began) return;

        if (raycastManager.Raycast(touch.position, hits,
            TrackableTypes.PlaneWithinPolygon))
        {
            Pose pose = hits[0].pose;
            if (placed == null)
                placed = Instantiate(prefab, pose.position, pose.rotation);
            else
                placed.transform.SetPositionAndRotation(
                    pose.position, pose.rotation);
        }
    }
}

Touch Input

Mobile AR uses touch, not mouse clicks. Every touch has a phase that tells you what the finger is doing right now.

Touch phases are like mouse events: Began = mousedown, Moved = mousemove, Ended = mouseup. Same idea, different device.

Touch phases:

PhaseMouse EquivalentWhen
TouchPhase.BeganmousedownFinger first touches screen
TouchPhase.Movedmousemove (while dragging)Finger slides on screen
TouchPhase.Stationary—Finger is down but not moving
TouchPhase.EndedmouseupFinger lifts off screen
TouchPhase.Canceled—System interrupted the touch
C# — Touch input handler 
void Update()
{
    if (Input.touchCount == 0) return;
    Touch touch = Input.GetTouch(0);

    switch (touch.phase)
    {
        case TouchPhase.Began:
            Debug.Log("Finger down at: " + touch.position);
            break;
        case TouchPhase.Moved:
            Debug.Log("Finger moving: " + touch.deltaPosition);
            break;
        case TouchPhase.Ended:
            Debug.Log("Finger lifted");
            break;
    }
}

06

Scale & Rotate

Two-finger gestures let users pinch to scale and twist to rotate placed objects. This measures the distance and angle between two touch points.

Like CSS transform: scale() and rotate() — but driven by finger gestures instead of code values.
C# — ScaleRotate.cs 
using UnityEngine;

public class ScaleRotate : MonoBehaviour
{
    public float scaleSpeed = 0.01f;
    public float rotateSpeed = 0.5f;

    void Update()
    {
        if (Input.touchCount == 2)
        {
            Touch t0 = Input.GetTouch(0);
            Touch t1 = Input.GetTouch(1);

            // Pinch to scale
            float prevDist = (t0.position - t0.deltaPosition
                - (t1.position - t1.deltaPosition)).magnitude;
            float currDist = (t0.position - t1.position).magnitude;
            float delta = currDist - prevDist;
            transform.localScale += Vector3.one * delta * scaleSpeed;
            transform.localScale = Vector3.Max(
                transform.localScale, Vector3.one * 0.1f);

            // Twist to rotate
            Vector2 prevDir = (t0.position - t0.deltaPosition)
                - (t1.position - t1.deltaPosition);
            Vector2 currDir = t0.position - t1.position;
            float angle = Vector2.SignedAngle(prevDir, currDir);
            transform.Rotate(Vector3.up, angle * rotateSpeed);
        }
    }
}

Multiple Objects

Let's level up: place multiple objects of different types, and add a UI button to switch between them. Each tap creates a new object.

Like an array of mesh constructors in Three.js. Pick which one to use, then scene.add() on each click.
C# — MultiPlacer.cs 
using System.Collections.Generic;
using UnityEngine;
using UnityEngine.XR.ARFoundation;
using UnityEngine.XR.ARSubsystems;

public class MultiPlacer : MonoBehaviour
{
    public GameObject[] prefabs;
    public int currentIndex = 0;

    ARRaycastManager raycastManager;
    static List<ARRaycastHit> hits = new List<ARRaycastHit>();
    List<GameObject> placed = new List<GameObject>();

    void Awake() => raycastManager =
        GetComponent<ARRaycastManager>();

    void Update()
    {
        if (Input.touchCount == 0) return;
        Touch touch = Input.GetTouch(0);
        if (touch.phase != TouchPhase.Began) return;

        if (raycastManager.Raycast(touch.position, hits,
            TrackableTypes.PlaneWithinPolygon))
        {
            var obj = Instantiate(prefabs[currentIndex],
                hits[0].pose.position, hits[0].pose.rotation);
            placed.Add(obj);
        }
    }

    public void NextPrefab() =>
        currentIndex = (currentIndex + 1) % prefabs.Length;

    public void ClearAll()
    {
        foreach (var obj in placed) Destroy(obj);
        placed.Clear();
    }
}

08

AR Raycast Deep Dive

There are two types of raycasts in AR, and mixing them up is a common mistake:

AR Raycast
Hits real-world tracked surfaces (planes, point clouds). Used to find where to PLACE new objects.

Physics Raycast
Hits Unity colliders on existing GameObjects. Used to SELECT already-placed objects.

AR Raycast = "where is the real floor?" Physics Raycast = "did I tap an existing cube?" You need BOTH for a complete AR experience.

The smart pattern: try Physics Raycast first (select existing). If nothing hit, try AR Raycast (place new).

C# — Combined raycast 
void Update()
{
    if (Input.touchCount == 0 ||
        Input.GetTouch(0).phase != TouchPhase.Began) return;

    Touch touch = Input.GetTouch(0);

    // First: did we tap an existing object?
    Ray ray = Camera.main.ScreenPointToRay(touch.position);
    if (Physics.Raycast(ray, out RaycastHit hit, 100f))
    {
        // Tapped an existing object - select it
        SelectObject(hit.transform.gameObject);
        return;
    }

    // Second: tap on AR plane to place new object
    if (raycastManager.Raycast(touch.position, hits,
        TrackableTypes.PlaneWithinPolygon))
    {
        PlaceNew(hits[0].pose);
    }
}

Toggle plane visibility — hide the grid once the user has placed objects:

C# — TogglePlanes.cs 
using UnityEngine;
using UnityEngine.XR.ARFoundation;

public class TogglePlanes : MonoBehaviour
{
    ARPlaneManager planeManager;
    bool visible = true;

    void Awake() => planeManager =
        FindObjectOfType<ARPlaneManager>();

    public void Toggle()
    {
        visible = !visible;
        foreach (var plane in planeManager.trackables)
            plane.gameObject.SetActive(visible);
        planeManager.enabled = visible;
    }
}

Build to Phone

This is the moment of truth. AR must run on a real device — no simulator can fake a phone camera and sensors.

You NEED a physical device. AR does not work in the Unity Editor simulator for beginners. No shortcuts here.

Android Build Steps:

1. File → Build Settings → Switch to Android
2. Player Settings → Company Name, Product Name
3. Minimum API Level: Android 7.0 (API 24)
4. Scripting Backend: IL2CPP
5. Target Architectures: ARM64

6. Connect phone via USB
7. Enable Developer Mode on phone
    Settings → About Phone → tap Build Number 7 times
8. Enable USB Debugging
    Settings → Developer Options → USB Debugging
9. Build And Run

iOS Build Steps:

1. File → Build Settings → Switch to iOS
2. Player Settings → Bundle Identifier
    com.yourname.arapp
3. Camera Usage Description:
    "Required for AR"

4. Click Build → opens Xcode project
5. In Xcode: sign with Apple Developer account
6. Connect iPhone, select as target
7. Build and run from Xcode
