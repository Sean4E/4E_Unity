00

Three.js → Unity

Everything you built in Three.js has a direct equivalent in Unity. The names change, but the ideas are the same. Here's the map:

Three.jsUnityWhat It Does
new THREE.Scene()Scene (Hierarchy window)Container for all objects
new THREE.Mesh()GameObject + MeshFilter + MeshRendererA visible 3D object
new THREE.BoxGeometry()GameObject → 3D Object → CubeA box shape
new THREE.MeshPhysicalMaterial()Material asset (Standard or URP/Lit)Surface appearance
new THREE.DirectionalLight()Light → Directional LightSun-like light
new THREE.PointLight()Light → Point LightLight bulb
new THREE.AmbientLight()Environment Lighting (Lighting window)Fill light everywhere
new THREE.PerspectiveCamera()Camera component on a GameObjectWhat the player sees
OrbitControlsPlayerController scriptCamera movement
GLTFLoaderDrag & drop into Assets folderImport 3D models
scene.backgroundSkybox material (Lighting settings)Background colour/image
requestAnimationFrameUnity's Update loop (automatic)Runs every frame
Think of Unity as "Three.js with a GUI". Instead of writing scene.add(mesh), you drag objects into the Hierarchy. Instead of editing code values, you adjust sliders in the Inspector. Same result, different workflow.
01

Create Your Project

Open Unity Hub and create a new project. Choose the 3D (Built-in Render Pipeline) template — it's the simplest and most compatible.

  1. Open Unity Hub → Click New Project
  2. Select 3D (Built-in Render Pipeline) template
  3. Name your project (e.g. InteractiveScene)
  4. Choose a save location
  5. Click Create project and wait for Unity to open

Organise Your Folders

In the Project window (bottom panel), right-click in the Assets folder and create these folders:

  1. Right-click in Assets → Create → Folder → name it Scripts
  2. Create another folder called Materials
  3. Create another folder called Models (for any imported 3D models)
  4. Create another folder called Audio (for any sound effects)

Import the Scripts

Download the 6 script files below and drag them into your Assets/Scripts folder in Unity's Project window.

💾PlayerController.cs
First-person movement 💾Interactable.cs
Press E detection 💾Door.cs
Open/close doors 💾Pickup.cs
Collect items + score 💾LightSwitch.cs
Toggle lights on/off 💾TriggerZone.cs
Area-based events
Wait for Unity to compile (the spinner in the bottom-right). If you see errors in the Console, check that all 6 files are in the Scripts folder.
Checkpoint: You have a Unity project with organised folders and 6 scripts ready to use. The Console should show no errors.
02

Build Your Environment

Just like you created shapes in Three.js with BoxGeometry and SphereGeometry, Unity has built-in primitives. Let's build a room.

Create the Floor

  1. Right-click in the Hierarchy → 3D Object → Plane
  2. Rename it to Floor (click the name in the Hierarchy)
  3. In the Inspector, set Position to X: 0, Y: 0, Z: 0
  4. Set Scale to X: 3, Y: 1, Z: 3 (makes a 30×30 metre floor)
In Three.js this was: new THREE.Mesh(new THREE.PlaneGeometry(30, 30), material) rotated to face up.

Create the Walls

  1. Right-click in Hierarchy → 3D Object → Cube
  2. Rename it to Wall_Back
  3. Set Position: X: 0, Y: 2, Z: 15
  4. Set Scale: X: 30, Y: 4, Z: 0.2
  5. Duplicate it (Ctrl+D) three times for the other walls

Position each wall:

Add Some Objects

Add at least 5 objects to fill your scene. Use primitives to prototype — you can replace them with imported models later:

  • Cube — table, crate, shelf, pillar, pedestal
  • Sphere — ball, decoration, orb
  • Cylinder — column, barrel, lamp post
  • Capsule — character placeholder, vase shape
Tip: Select an object and press F to focus the Scene camera on it. Hold right-click + WASD to fly around the Scene view.

Organise with Empty Parents

Keep your Hierarchy tidy by grouping objects:

  1. Right-click in Hierarchy → Create Empty
  2. Rename it to Environment
  3. Drag all your walls and floor onto it (they become children)
  4. Create another empty called Props for your objects

03

Add the Player

In Three.js you had OrbitControls for camera movement. In Unity, we create a Player object with a camera attached and the PlayerController script for first-person controls.

  1. Delete the default Main Camera from the Hierarchy (right-click → Delete)
  2. Right-click in Hierarchy → Create Empty
  3. Rename it to Player
  4. Set Position to X: 0, Y: 0, Z: -10 (near the front wall, inside the room)
  5. With Player selected, click the Tag dropdown (top of Inspector) → choose Player

Add Components to Player

  1. Click Add Component → search Character Controller → add it
  2. Set Character Controller settings:
    • Center: X: 0, Y: 0.9, Z: 0
    • Radius: 0.3
    • Height: 1.8
  3. Click Add Component → search PlayerController → add it

Add the Camera

  1. Right-click on Player in the Hierarchy → Camera
  2. A Camera appears as a child of Player
  3. Set the Camera's local Position to X: 0, Y: 1.6, Z: 0 (eye height)
  4. Make sure local Rotation is 0, 0, 0
  5. Important: Select the Camera, click the Tag dropdown at the top of the Inspector, and set it to MainCamera. Without this tag, the interaction scripts cannot detect where you are looking!

04

Materials & Colours

In Three.js you created materials with new THREE.MeshPhysicalMaterial({ color: 0x8b5cf6, metalness: 0.3, roughness: 0.5 }). In Unity, you create Material assets and drag them onto objects.

Create a Material

  1. In the Project window, navigate to your Materials folder
  2. Right-click → Create → Material
  3. Name it (e.g. Floor_Material)
  4. Click the material to see its properties in the Inspector
  5. Click the Albedo colour swatch to pick a colour
  6. Adjust Metallic slider (0 = plastic, 1 = metal)
  7. Adjust Smoothness slider (0 = rough, 1 = mirror-like)
  8. Drag the material from the Project window onto the object in the Scene view
Three.js PropertyUnity PropertyValues
colorAlbedoColour picker
metalnessMetallic0 to 1 slider
roughnessSmoothness (inverse!)0 = rough, 1 = smooth
emissiveEmission (tick checkbox)Colour + intensity
Key difference: Three.js uses roughness (high = rough). Unity uses Smoothness (high = smooth). They're inverses — a roughness of 0.3 in Three.js is a Smoothness of 0.7 in Unity.

Create Multiple Materials

Create at least 4 different materials for variety:

  • Floor — dark colour, low smoothness (concrete/wood look)
  • Walls — neutral colour, low smoothness
  • Metal — grey, high metallic, medium smoothness
  • Accent — a bright colour for key objects (doors, pickups)
Tip: You can drag a material onto multiple objects. One material can be shared by many objects, just like in Three.js.
Checkpoint: Your scene has coloured objects. No more default grey. At least 4 different materials applied.
05

Lighting & Atmosphere

In Three.js you added lights with new THREE.DirectionalLight() and new THREE.PointLight(). Unity works the same way — but you create light objects in the scene instead of writing code.

Set Up Your Lights

Your scene already has a Directional Light (Unity creates one by default). Let's configure it and add more:

  1. Select the existing Directional Light in the Hierarchy
  2. Set its Rotation to X: 50, Y: -30, Z: 0 (angle of the sun)
  3. Set Intensity to 1
  4. Tick Shadow Type → Soft Shadows

Add Point Lights

Point lights are like new THREE.PointLight() — they emit light in all directions from a point.

  1. Right-click in Hierarchy → Light → Point Light
  2. Rename it (e.g. RoomLight_1)
  3. Position it where you want a light source (above a table, in a corner)
  4. Set Range (how far the light reaches)
  5. Set Intensity and Color
  6. Add 2-3 point lights around your scene for atmosphere
Three.jsUnityNotes
DirectionalLight(0xffffff, 1.0)Directional Light, Intensity: 1Sun/moon — parallel rays
PointLight(0xffa500, 1, 10)Point Light, Range: 10Bulb — light in all directions
SpotLight()Spot LightFlashlight / stage light
AmbientLight(0xffffff, 0.3)Window → Lighting → EnvironmentGlobal fill light

Set the Background Colour

In Three.js: scene.background = new THREE.Color(0x1a1a2e). In Unity:

  1. Select the Camera (child of Player)
  2. In the Inspector, find Clear Flags → set to Solid Color
  3. Click the Background colour to pick your sky colour
Alternative: Use Window → Rendering → Lighting → Environment tab to set a Skybox material for a sky image instead of a flat colour.
Checkpoint: Your scene has mood. At least one directional light with shadows and 2-3 point lights. The background isn't default blue.

Making It Interactive

06

Add Interactions

You need at least 3 different interaction types for your assignment. Here's how to set up each one using the scripts we imported. No coding — just drag and drop in the Inspector.

Interaction 1: Door

A door that opens and closes when the player presses E.

  1. Create a Cube for the door (Scale: 1, 2, 0.1)
  2. Create an Empty GameObject — rename it DoorPivot
  3. Position DoorPivot at the hinge edge of the door
  4. Make the door Cube a child of DoorPivot
  5. Offset the door's local X position by half its width (e.g. 0.5) so the edge sits at the pivot
  6. Select DoorPivot and add these components:
    • Box Collider (adjust size to cover the door)
    • Interactable — set Prompt Text: Press E to open
    • Door
  7. In the On Interact event: click +, drag DoorPivot into the object slot, select Door → ToggleDoor

Interaction 2: Pickup

A floating, spinning gem that disappears when collected.

  1. Create a Sphere (or any shape for your collectible)
  2. Scale it down: 0.3, 0.3, 0.3
  3. Apply a bright material (gold, green, etc.)
  4. Position it floating above a surface (e.g. Y: 1.5)
  5. Add components:
    • Sphere Collider (should auto-add)
    • Interactable — Prompt: Press E to collect
    • Pickup
  6. In On Interact: drag this object, select Pickup → Collect
  7. Duplicate (Ctrl+D) and scatter copies around your scene

Interaction 3: Light Switch

A switch on the wall that toggles a nearby light.

  1. Create a small Cube on a wall (Scale: 0.2, 0.3, 0.05) — this is the switch
  2. Make sure there's a Point Light nearby that this switch will control
  3. Add components to the switch:
    • Box Collider
    • Interactable — Prompt: Press E to toggle light
    • LightSwitch
  4. Drag the Point Light from the Hierarchy into the Target Light slot
  5. In On Interact: drag the switch object, select LightSwitch → ToggleLight

Bonus: Trigger Zone

An invisible area that fires events when the player walks through — no button press needed.

  1. Create an Empty GameObject — rename it (e.g. EntryTrigger)
  2. Add a Box Collider
  3. Tick Is Trigger on the Box Collider
  4. Resize the collider to cover a doorway or area (Edit Collider button or Size fields)
  5. Add the TriggerZone script
  6. In On Player Enter: click +, drag a Light, select Light → enabled → tick the checkbox
Remember: Your Player must have the "Player" tag set, or triggers won't detect it.
Checkpoint: You have at least 3 working interactions. Test each one in Play mode.
07

Polish & Import Models

Replace prototype cubes with real 3D models. In Three.js you used GLTFLoader — in Unity, you just drag and drop.

Import GLB/FBX Models

  1. Drag your .glb, .fbx, or .obj file into the Assets/Models folder in the Project window
  2. Unity imports it automatically
  3. Click the imported model → Inspector → Materials tab → click Extract Textures and Extract Materials to get editable materials
  4. Drag the model from the Project window into the Scene
  5. Position and scale it to fit your scene
In Three.js: loader.load('model.glb', (gltf) => scene.add(gltf.scene)). In Unity: just drag it in. Same result.

Free Asset Sources

  • Your own Blender models from earlier weeks (export as GLB or FBX)
  • Sketchfab — filter by "Downloadable" and CC licence
  • Unity Asset Store — many free packs available
  • Poly Pizza — free low-poly models
  • Kenney.nl — free game assets (CC0)

Add Sound Effects

  1. Drag .wav or .mp3 files into your Assets/Audio folder
  2. For pickup sounds: drag the audio clip into the Pickup script's Pickup Sound slot
  3. For ambient sounds: add an Audio Source component to any object, assign the clip, tick Loop and Play On Awake
Checkpoint: Your scene looks polished with imported models and/or textured primitives. Optionally has sound.
08

Build & Export

Time to create a standalone application that anyone can run — no Unity required.

Build Steps

  1. Go to File → Build Settings
  2. Click Add Open Scenes to include your current scene
  3. Select Windows, Mac, Linux as the platform (or WebGL)
  4. Click Player Settings → set your Product Name
  5. Click Build
  6. Choose a folder for the output (create a new folder called Build)
  7. Wait for Unity to build your application

Test Your Build

  1. Navigate to the Build folder
  2. Run the .exe file
  3. Test all interactions work
  4. Check for any errors or missing textures
  5. Zip the entire Build folder for submission
Important: When zipping for submission, include ALL files in the Build folder. The .exe alone won't work — it needs the Data folder next to it.

Submission Files

  • Lastname_Firstname_W9_Build.zip — the built application
  • Lastname_Firstname_W9_Screenshot.png — screenshot of your scene
  • Lastname_Firstname_W9_Demo.mp4 — screen recording (60-90 seconds)
  • Lastname_Firstname_W9_Project.zip — your Unity project folder
Checkpoint: Your build runs outside of Unity. All interactions work. You have your submission files ready.

  • No errors in the Console

Three.js ↔ Unity Comparison

A side-by-side look at how the same concepts work in both environments. Use this as a quick reference.

Three.js (Code)

// Create a red box
const box = new THREE.Mesh(
  new THREE.BoxGeometry(1,1,1),
  new THREE.MeshPhysicalMaterial({
    color: 0xff0000,
    metalness: 0.5,
    roughness: 0.3
  })
);
box.position.set(0, 0.5, 0);
scene.add(box);

// Add a point light
const light = new THREE.PointLight(
  0xffa500, 1, 10
);
light.position.set(2, 3, 0);
scene.add(light);

// Load a GLB model
const loader = new GLTFLoader();
loader.load('model.glb', (gltf) => {
  scene.add(gltf.scene);
});

Key Differences

AspectThree.jsUnity
WorkflowWrite code, refresh browserDrag and drop, press Play
Animation loopYou write requestAnimationFrameAutomatic (Update runs every frame)
PhysicsManual or add a libraryBuilt-in (Rigidbody + Colliders)
InteractionsCode raycasting manuallyScript + Inspector events
OutputHTML file (runs in browser).exe / WebGL build
PerformanceLimited by browserNative performance, GPU access
File sizeSingle HTML (small)Build folder (larger)
Both tools create 3D interactive experiences. Three.js lives in the browser and uses code. Unity is a full engine with a visual editor. The 3D concepts — meshes, materials, lights, cameras, transforms — are identical. Once you understand one, you understand both.
